Subsystems

AbsoluteLib provides pre-built, production-ready, and simulation-compatible classes for common FRC mechanisms. These subsystems simplify control, motion profiling, and physics simulation.

Pivot

The Pivot class controls a single-joint rotating arm (e.g., wrist, intake arm, simple shooter angle). It supports ProfiledPID control, feedforward, stiffness simulation, and Smart Motion.

Configuration

public class Wrist extends Pivot {
    public Wrist() {
        super(new Pivot.Config()
            // Hardware
            .withLeader(new MotorWrapper(MotorType.TALONFX, 15))
            .withEncoder(EncoderWrapper.canCoder(20, 0.0)) // ID 20, offset 0.0
            .encoderInverted(false)
            .gear(50.0) // 50:1 gear ratio

            // Limits & Motion
            .limits(Math.toRadians(-90), Math.toRadians(90))
            .tolerance(Math.toRadians(1.0))

            // Control (PID + Feedforward)
            .pid(0.02, 0.0, 0.0)
            .ff(0.0, 0.45, 0.0, 0.0) // kS, kG, kV, kA

            // Power Distribution & Motion Magic
            .motion(Math.toRadians(180), Math.toRadians(360)) // Max Vel, Max Accel

            // Simulation
            .enableSimulation(true)
            .withSimulation(new PivotSimulation.PivotSimulationConfig()
                .armLength(0.35) // meters
                .armMass(2.5)    // kg
                .limits(Math.toRadians(-95), Math.toRadians(95))
            )
        );
    }
}

Usage

Wrist wrist = new Wrist();

// Set target angle in radians
wrist.setTargetAngle(Math.toRadians(45));

// Create a command to move to position
Command moveCmd = wrist.setAngle(Math.toRadians(90));

Elevator

The Elevator class controls a linear actuator. It handles gravity feedforward, position limits, and motion profiling.

Configuration

public class Lift extends Elevator {
    public Lift() {
        super(new Elevator.Config()
            .withLeader(new MotorWrapper(MotorType.SPARKMAX, 10))
            .withFollowers(new MotorWrapper(MotorType.SPARKMAX, 11))
            .gear(10.0)
            .drumRadius(0.05) // 5cm radius drum
            .limits(0.0, 1.5) // 0 to 1.5 meters

            // Control
            .pid(5.0, 0.0, 0.0)
            .ff(0.0, 0.1, 2.5, 0.1) // kG, kV, kA (kS usually 0 for elevator)

            // Simulation
            .enableSimulation(true)
            .withSimulation(new ElevatorSimulation.ElevatorSimulationConfig()
                .carriageMass(4.0) // kg
                .minHeight(0.0)
                .maxHeight(1.55)
            )
        );
    }
}

Arm (Multi-Joint)

The Arm class manages multi-jointed arms using Inverse Kinematics (IK). It allows you to control the end-effector in Cartesian space (x, y) rather than managing individual joint angles.

Initialization

public class SuperStructure extends AbsoluteSubsystem {
    private final Arm arm = new Arm();
    private final Arm.Joint shoulder;
    
    public SuperStructure() {
        // 1. Configure the Motor
        MotorWrapper shoulderMotor = new MotorWrapper(MotorType.TALONFX, 1);
        shoulderMotor.setInverted(true);
        
        // PID Configuration via MotorWrapper or MotorConfig
        MotorWrapper.UnifiedMotorConfig motorCfg = MotorWrapper.UnifiedMotorConfig.builder()
            .kP(0.5)
            .kI(0.0)
            .kD(0.0)
            .build();

        // 2. Configure the Joint Kinematics
        Arm.JointConfig shoulderKinematics = Arm.JointConfig.builder()
            .linkLengthMeters(0.8) // Arm segment length
            .minAngleRad(Math.toRadians(-90))
            .maxAngleRad(Math.toRadians(90))
            .metersToRadians(1.0) // If encoder returns radians directly
            .build();
        
        // 3. Add Joint to Arm
        shoulder = arm.addJoint(
            shoulderMotor,
            motorCfg, // Apply PID config
            EncoderWrapper.canCoder(1, 0.0), // Encoder
            shoulderKinematics
        );

        // ... Add Elbow Joint ...

        arm.initialize(); // Must call after adding joints
    }
}

Cartesian Control

// Move end-effector to x=0.5m, y=1.2m
arm.setGoalPose(new Translation2d(0.5, 1.2)); // or arm.setGoalPose(0.5, 1.2)

// Periodic update runs the IK solver and PID loops
@Override
public void periodic() {
    arm.periodic();
}

End Effector

A simple class for intake/outtake mechanisms. It handles leader/follower setup and simple speed control.

public class Intake extends AbsoluteSubsystem {
    private final EndEffector roller;

    public Intake() {
        roller = new EndEffector(new EndEffector.Config()
            .withLeader(new MotorWrapper(MotorType.SPARKMAX, 30))
            .withFollowers(new MotorWrapper(MotorType.SPARKMAX, 31)) // Optional follower
            .inverted(false)
        );
    }

    public void intake() {
        roller.start(0.8);
    }

    public void eject() {
        roller.start(-0.8);
    }
    
    public void stop() {
        roller.stop();
    }
}
