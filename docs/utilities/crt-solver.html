CRT Solver & Turret Anti-Windup

The ChineseRemainderSolver is a general-purpose math utility in the ca.team4308.absolutelib.math package. It provides two main capabilities:

  • Chinese Remainder Theorem (CRT): Solve systems of modular congruences to find values satisfying multiple discrete constraints simultaneously.
  • Turret Anti-Windup: Find the shortest rotation to a target heading that stays within cable/slip-ring limits — no over-rotation.

Turret Anti-Windup (Primary Use Case)

When a turret can rotate continuously but has cable limits (e.g., ±1.5 revolutions), you need to find the closest equivalent heading that doesn't exceed those limits. The solver handles this automatically.

import ca.team4308.absolutelib.math.ChineseRemainderSolver;

// Configuration
int TICKS_PER_REV = 4096;       // encoder resolution
double MIN_TICKS = -6144;       // -1.5 revolutions (CCW cable limit)
double MAX_TICKS = 6144;        // +1.5 revolutions (CW cable limit)

// In your turret periodic():
double currentTicks = turretEncoder.getPosition();
double targetDegrees = limelightYaw; // e.g., 270°

// One call does it all — finds shortest path within limits
double safeDest = ChineseRemainderSolver.safeTurretDestination(
    currentTicks, targetDegrees, TICKS_PER_REV, MIN_TICKS, MAX_TICKS);

turretMotor.setPosition(safeDest);

Step-by-Step Breakdown

If you need more control, use the individual methods:

// 1. Convert target heading to ticks
double targetTick = ChineseRemainderSolver.degreesToTicks(270.0, 4096);
// → 3072.0

// 2. Find shortest rotation (wraps around one revolution)
double shortestMove = ChineseRemainderSolver.shortestWrappedMove(
    currentTicks, targetTick, 4096);
// → could be +500 or -3596 — picks the shorter one

// 3. Check if the ideal destination is within cable limits
double destination = currentTicks + shortestMove;

// 4. If it exceeds limits, clamp safely
double safeDest = ChineseRemainderSolver.clampToWindupLimits(
    currentTicks, targetTick, 4096, -6144, 6144);
// → picks the closest valid equivalent position

Discrete Value Snapping

Snap continuous RPM or angle values to hardware-achievable steps:

// Snap to nearest 100 RPM
double snappedRpm = ChineseRemainderSolver.snapToResolution(3487, 100);
// → 3500.0

// Snap to nearest 0.1°
double snappedAngle = ChineseRemainderSolver.snapToResolution(45.47, 0.1);
// → 45.5

Core CRT — Modular Constraints

For advanced use: solve systems of modular congruences. For example, find a position that satisfies both encoder tick alignment AND gear mesh alignment:

import ca.team4308.absolutelib.math.ChineseRemainderSolver.*;

// x ≡ 2 (mod 3) AND x ≡ 3 (mod 5) AND x ≡ 2 (mod 7)
List<Constraint> constraints = List.of(
    new Constraint("encoder", 2, 3),
    new Constraint("gear-A", 3, 5),
    new Constraint("gear-B", 2, 7)
);

Solution sol = ChineseRemainderSolver.solve(constraints);
// sol.getValue() = 23, sol.getPeriod() = 105

// Find closest valid position to where we are now
long nearest = sol.nearest(currentPosition);

// Enumerate all valid positions in a range
List<Long> options = sol.solutionsInRange(0, 500);

Number Theory Utilities

long g = ChineseRemainderSolver.gcd(12, 18);  // 6
long l = ChineseRemainderSolver.lcm(12, 18);  // 36
long[] ext = ChineseRemainderSolver.extendedGcd(12, 18);
// ext = [6, -1, 1] → 12*(-1) + 18*(1) = 6

Legacy API

The old class names ModularConstraint and CRTSolution still work as subclass aliases of Constraint and Solution:

// Both work:
new ChineseRemainderSolver.Constraint("enc", 100, 4096);
new ChineseRemainderSolver.ModularConstraint("enc", 100, 4096);
