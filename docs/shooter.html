Shooter System

The ShooterSystem is the high-level manager that integrates the TrajectorySolver, ShotLookupTable, and hardware control. It handles state machines, safety validation, movement compensation, and fallback references.

Shot Modes

  • LOOKUP_ONLY: Uses a static lookup table (fastest, most reliable for static shots).
  • SOLVER_ONLY: Uses the physics solver for every frame (most flexible, higher CPU usage).
  • LOOKUP_WITH_SOLVER_FALLBACK: Tries lookup table first; if out of range, uses solver.
  • SOLVER_WITH_LOOKUP_FALLBACK: Tries solver first; if it fails (e.g., no solution), uses lookup table.
  • BLENDED: Interpolates between solver and lookup table for smooth transitions. Uses Source.BLENDED to prevent double movement compensation.

Lookup Table Setup

The ShotLookupTable uses InterpolatingDoubleTreeMap for smooth interpolation between data points. Distances are automatically clamped to the table bounds.

// Build lookup table from practice data
ShotLookupTable lookup = new ShotLookupTable();
lookup.addEntry(2.0, 55.0, 3000);           // 2m: 55째, 3000 RPM
lookup.addEntry(3.0, 45.0, 3500);           // 3m: 45째, 3500 RPM
lookup.addEntry(4.0, 38.0, 4200, 0.45);     // 4m: 38째, 4200 RPM, 0.45s TOF
lookup.addEntry(5.5, 30.0, 5000, 0.62);     // 5.5m: 30째, 5000 RPM, 0.62s TOF

// For custom wheel diameters, pass an RPM-to-velocity factor
ShotLookupTable customLookup = new ShotLookupTable(0.006); // different wheel size

Example Implementation

// Initialize Dependencies
ShooterConfig config = new ShooterConfig();
ShotLookupTable lookup = new ShotLookupTable();
// ... add entries ...
TrajectorySolver solver = TrajectorySolver.builder()
    .gamePiece(GamePieces.REBUILT_2026_BALL)
    .config(SolverConfig.defaults())
    .build();

// Create System
ShooterSystem system = new ShooterSystem(config, lookup, solver);
system.setMode(ShotMode.SOLVER_WITH_LOOKUP_FALLBACK);

// Periodic Loop
public void periodic() {
    ShotParameters shot = system.calculate(
        distanceToTarget,
        currentFlywheelRpm,
        robotVx, robotVy,
        yawToTarget
    );

    if (shot.isValid()) {
        pivot.setPosition(shot.pitchDegrees);
        flywheel.setVelocity(shot.rpm);
        
        if (system.isReadyToFire(currentFlywheelRpm) && pivot.isAtSetpoint()) {
            feeder.run();
        }
    } else {
        pivot.setPosition(config.stowAngle);
    }
}

Movement Compensation

The MovementCompensator automatically adjusts shots for robot movement. In v2.1.0 this includes:

  • NaN guard: Non-finite exit velocities are safely handled by returning the base shot unchanged.
  • Yaw accumulation: Movement yaw lead is added to the solver's existing yaw correction (spin drift, etc.) instead of replacing it.
  • Blended source awareness: Blended shots use Source.BLENDED to avoid double-compensation.

Safety Features

The system includes a SafetyValidator that prevents:

  • Shots from too far away (max distance).
  • Shots with impossible RPM or angles.
  • Firing when the robot is moving too fast (if configured).

Flywheel Integration

The library includes a FlywheelGenerator for simulating thousands of flywheel configurations and a FlywheelSimulator for spin-up analysis.

// Configure flywheel hardware
FlywheelConfig flywheel = new FlywheelConfig(
    FlywheelConfig.FlywheelType.DUAL_OVER_UNDER,
    GamePieces.REBUILT_2026_BALL,
    4.0,  // wheel diameter inches
    FRCMotors.KRAKEN_X60,
    1.0   // gear ratio
);
solver.setFlywheel(flywheel);

// Generate optimal configurations
FlywheelGenerator generator = new FlywheelGenerator(
    GamePieces.REBUILT_2026_BALL,
    GenerationParams.detailed()
);
GenerationResult result = generator.generateAndEvaluate(25.0); // 25 m/s target
