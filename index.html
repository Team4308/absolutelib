Introduction

AbsoluteLib is a comprehensive FRC utility library developed by Team 4308. It unifies hardware interaction, simplifies simulation, and provides robust subsystems for common robot mechanisms.

Installation

To install AbsoluteLib in your WPILib project:

  1. Open your project in VS Code.
  2. Press Ctrl+Shift+P and select WPILib: Manage Vendor Libraries.
  3. Select Install new library (online).
  4. Enter the following URL:
https://team4308.github.io/absolutelib/lib/absolutelib.json

Unified Motor Wrapper

Control TalonFX, TalonSRX, VictorSPX, and SparkMax motors with a single, unified API. No more vendor-specific boilerplate.

Basic Usage

MotorWrapper shooter = new MotorWrapper(MotorWrapper.MotorType.TALONFX, 10);
shooter.setVoltage(12.0);

Smart Motion Configuration

Only works with Sparkmaxs and TalonSX, TalonSRX'S Dont support Motion Magic.
shooter.applyMotorConfig(MotorWrapper.UnifiedMotorConfig.builder()
    .kP(0.1)
    .kF(0.05)
    .motionCruiseVelocity(6000)
    .motionAcceleration(12000)
    .build());

shooter.setSmartPosition(10.0); // Moves 10 rotations

Pivot Subsystem

The Pivot class provides a robust base for Arms, Wrists, and Elevators with built-in physics simulation.

public class Wrist extends Pivot {
    public Wrist() {
        super(new Config()
            .withLeader(new MotorWrapper(MotorWrapper.MotorType.TALONFX, 15))
            .withEncoder(EncoderWrapper.canCoder(20, 0.0))
            .pid(0.02, 0.0, 0.0)
            .enableSimulation(true)
            .withSimulation(new PivotSimulation.Config()
                .armLength(0.3)
                .armMass(2.0)
            )
        );
    }
}

Trajectory System (2026 REBUILT)

Complete trajectory solving system for turret shooters with flywheel simulation, projectile physics, and shot optimization.

Quick Start

// Create solver for 2026 game
TrajectorySolver solver = TrajectorySolver.forGame2026();

// Define shot
ShotInput input = ShotInput.builder()
    .shooterPositionMeters(1.0, 2.0, 0.5)
    .targetPositionMeters(5.0, 5.0, 2.5)
    .shotPreference(ShotInput.ShotPreference.FASTEST)
    .build();

// Get best pitch angle
double pitch = solver.solveBestPitchDegrees(input);
System.out.println("Shoot at: " + pitch + "°");

Multi-Candidate Selection

// Find all possible shots
ShotCandidateList candidates = solver.findAllCandidates(input);

// Get shots by preference
candidates.getFastest();        // Fastest shot
candidates.getMostAccurate();   // Most accurate
candidates.getMostStable();     // Most stable angle
candidates.getMaxClearance();   // Highest arc

Flywheel Configuration

// Generate optimal flywheel
FlywheelGenerator generator = new FlywheelGenerator(
    GamePieces.REBUILT_2026_BALL
);

FlywheelGenerator.GenerationResult result = 
    generator.generateAndEvaluate(15.0); // 15 m/s target

FlywheelConfig best = result.bestConfig.config;
System.out.println("Best: " + best.toString());

Key Features

  • Physics-Based: Accurate projectile motion with air resistance and Magnus effect
  • Flywheel Simulation: Realistic wheel-ball interaction modeling
  • Auto-Optimization: Generates and scores hundreds of configurations
  • Multi-Candidate: Returns all valid shots sorted by preference
  • 2026 REBUILT: Optimized for 5.91" high-density foam ball

Maintenance & Releases

Guide for maintainers on how to update and release new versions of AbsoluteLib.

Version Bump Workflow

  1. Update Gradle Properties:

    Open gradle.properties and increment the version number.

  2. Update Vendor JSON:

    Open absolutelib.json and ensure the "version" field matches the new version in gradle.properties.

  3. Update Documentation:

    If APIs have changed, update the README.md examples and this documentation site.

Publishing a Release

We use an automated script to handle tagging, building, and publishing.

# Run the release script from the project root
./scripts/publish.bat

This script will:

  • Build the library.
  • Commit changes with the new version number.
  • Push changes to GitHub
# Run the site script from the project root
                    ./scripts/build-pages-site.bat

This script will:

  • Build the website.
  • Commit changes with the new version number. to /gh-pages
